Why it's hard (and interesting)

A US Amazon card is useless in Lagos. Cross-border value delivery is an optimization + fulfillment problem, not a lookup: pick the right instruments for the destination, fit them to real denominations and a budget, then settle — autonomously. Conduit does the whole loop over Bitrefill's catalog of 1,500+ brands across 170+ countries.

How it works

1 Intent — an LLM parses “$40 to my brother in Lagos” into amount, destination (NG), and needs.  →  2 Discover — pull the country's real instruments (airtime, local gift cards, eSIM, bills) from the Bitrefill REST API.  →  3 Optimize — the agent allocates the budget across the most locally-useful mix; code snaps each share to a real denomination by its true USD cost.  →  4 Policy — spend caps replace a human approval click, so it's truly autonomous.  →  5 Checkout — one multi-item invoice, paid from balance, each order polled to delivered and the redemption codes returned.

Autonomous checkout

One POST /invoices with many products, payment_method: balance, auto_pay. No human in the loop.

Real cost-aware routing

Budgets against each package's true settlement price, not face value — so cross-currency math is honest.

Per-order delivery

Polls orders[i].status (never the flaky rollup) and masks every redemption code.

Spend policy

Per-route / per-order caps and country allow-lists gate every purchase — autonomy with guardrails.

Run it

uv sync
cp .env.example .env   # BITREFILL_API_KEY + a free LLM key (LLM_API_KEY)
uv run uvicorn web.app:app --reload    # open http://localhost:8000

# or headless:
uv run python -m bitrefill_agent.router.engine "Get $40 to my brother in Lagos, Nigeria"

Demos run in safe mode on a free Bitrefill test product (delivers real codes, spends nothing). --live spends provisioned test credits on the real basket.
